

## 1. Открытие Swagger

В браузере открыть:

```text
http://localhost:8080/swagger-ui/index.html
```

- Swagger UI открылся;
- видны группы эндпоинтов: Auth, Buildings, Rooms, Sensors, Readings, Alerts, Incidents, Reports, Import, Telegram.

---

## 2. Авторизация в Swagger

В Swagger выполнить запрос:

```http
POST /api/auth/login
```

Тело запроса:

```json
{
  "username": "admin",
  "password": "admin123"
}
```

- получен JWT-токен;
- токен скопирован;
- нажать кнопку `Authorize` в Swagger;
- вставить токен в формате:

```text
Bearer <полученный_токен>
```

---

## 3. Проверка GET-запросов

Проверить список корпусов:

```http
GET /api/buildings
```

Проверить список помещений:

```http
GET /api/rooms
```

Проверить список датчиков:

```http
GET /api/sensors
```

- запросы выполняются успешно;
- сервер возвращает JSON-ответ;
- видно, что данные уже есть после инициализации.

---

## 4. Создание корпуса

В Swagger выполнить:

```http
POST /api/buildings
```

Пример тела запроса:

```json
{
  "name": "Учебный корпус №1",
  "address": "г. Набережные Челны",
  "description": "Основной учебный корпус"
}
```

Потом проверить:

```http
GET /api/buildings
```

- объект создан;
- новый корпус появился в списке.

---

## 6. Создание датчика

В Swagger выполнить:

```http
POST /api/sensors
```

Пример тела запроса:

```json
{
  "inventoryNumber": "SMOKE-204",
  "type": "SMOKE",
  "status": "ACTIVE",
  "roomId": 1,
  "thresholdValue": 50.0
}
```

Потом проверить:

```http
GET /api/sensors
```

- датчик создан;
- у датчика есть тип, статус, помещение и пороговое значение.

---

## 7. Создание показания датчика без тревоги

В Swagger выполнить:

```http
POST /api/readings
```

Пример тела запроса:

```json
{
  "sensorId": 1,
  "value": 25.0
}
```

- показание сохранено;
- тревога не создаётся, потому что значение ниже порога.

---

## 8. Создание показания с превышением порога

В Swagger выполнить:

```http
POST /api/readings
```

Пример тела запроса:

```json
{
  "sensorId": 1,
  "value": 80.0
}
```

- показание сохранено;
- автоматически создана тревога;
- в логах видно событие;
- в Telegram пришло уведомление от бота.

---

## 9. Проверка тревог

В Swagger выполнить:

```http
GET /api/alerts
```

- в списке появилась новая тревога;
- у тревоги есть датчик, сообщение, статус и дата создания.

---

## 10. Изменение статуса тревоги

В Swagger выполнить:

```http
PUT /api/alerts/{id}/status
```

Пример параметра:

```text
status=IN_PROGRESS
```

Потом можно поставить:

```text
status=RESOLVED
```

- статус тревоги изменяется;
- система работает не только как CRUD, но и как реальный процесс обработки события.

---

## 12. Загрузка файла к инциденту

В Swagger выполнить:

```http
POST /api/incidents/{incidentId}/photos
```

Действие:
- выбрать файл изображения;
- нажать Execute.

- файл загружен;
- информация о файле сохранена;
- в проекте появилась папка `uploads`.

---

## 13. Импорт датчиков из CSV

В Swagger выполнить:

```http
POST /api/import/sensors/csv
```

Файл для примера:

```text
docs/examples/sensors.csv
```

- CSV-файл загружен;
- датчики импортированы;
- результат импорта сохранён в ImportLog.

Потом проверить:

```http
GET /api/sensors
```
