## Введение

В рамках курса «Кроссплатформенное программное обеспечение» был разработан backend REST API для системы мониторинга пожарной безопасности учебного корпуса. Система реализует полный цикл учёта объектов инфраструктуры, обработку событий датчиков, управление тревогами и инцидентами, интеграцию с Telegram и формирование отчётов.

Технология Java+Spring Boot была выбрана как наиболее распространённая в промышленной разработке enterprise-систем, обеспечивающая кроссплатформенность через JVM.

## Что изучено на курсе

### 1. Java 17 и система сборки Maven

Java — объектно-ориентированный язык с платформенной независимостью. Байт-код выполняется на любой JVM (Windows, Linux, macOS). В Java 17 использованы: records, sealed classes, switch expressions, text blocks.

Maven — декларативный инструмент сборки. Файл `pom.xml` описывает зависимости, плагины и жизненный цикл сборки. Команда `mvn spring-boot:run` компилирует и запускает приложение.

**Применение в проекте:** все зависимости (Spring Boot, JWT, POI, OpenPDF) подключены через `pom.xml`. Структура проекта соответствует стандарту Maven.

### 2. HTTP и REST-архитектура

REST — архитектурный стиль для создания веб-сервисов. Основан на HTTP-методах: GET (чтение), POST (создание), PUT (обновление), DELETE (удаление). Важный принцип — idempotency: повторный PUT с теми же данными даёт тот же результат.

**Применение:** API предоставляет 12 групп endpoint'ов. Все ресурсы доступны по адресам вида `/api/{resource}/{id}`. Сервер возвращает JSON через Jackson.

### 3. DI/IoC — Dependency Injection

IoC-контейнер Spring управляет созданием и уничтожением объектов. Зависимости внедряются через конструктор (`@RequiredArgsConstructor`), а не создаются вручную. Это упрощает тестирование: зависимости легко заменяются на моки.

**Применение:** `SensorReadingService` получает `SensorRepository` и `AlertService` через конструктор. В тестах используется `@Mock` для замены реальных реализаций.

### 4. Spring Web MVC

`@RestController` + `@RequestMapping` создают REST-контроллеры. `@PathVariable` извлекает ID из URL, `@RequestParam` — параметры запроса. `@RequestBody` десериализует JSON в DTO, `@Valid` запускает валидацию.

**Применение:** 12 контроллеров обрабатывают все HTTP-запросы. Глобальный `@ControllerAdvice` перехватывает исключения и возвращает структурированные ответы.

### 5. Spring Data JPA и PostgreSQL

JPA (Java Persistence API) — стандарт ORM. `@Entity` описывает таблицу, `@JpaRepository` предоставляет готовые CRUD-методы. `@Transactional` гарантирует ACID-свойства операций.

**Применение:** 13 сущностей с аннотациями `@OneToMany`, `@ManyToOne`, `@ManyToMany`. Для предотвращения N+1 problem используется `@EntityGraph`. `FetchType.LAZY` снижает нагрузку на БД.

### 6. Spring Security и JWT

Spring Security обеспечивает аутентификацию (кто ты?) и авторизацию (что можешь?). JWT (JSON Web Token) — stateless-токен, содержащий claims пользователя. BCrypt — адаптивная хэш-функция для хранения паролей.

**Применение:**
- `SecurityConfig` настраивает stateless политику
- `JwtAuthenticationFilter` проверяет токен на каждый запрос
- `@PreAuthorize("hasAuthority('SENSOR_DELETE')")` — проверка прав доступа
- Пароли хешируются BCrypt перед сохранением

### 7. Работа с файлами

Загрузка файлов — через `multipart/form-data`. MIME-тип проверяется для безопасности: без проверки злоумышленник может загрузить скрипт вместо изображения.

**Применение:** `FileStorageService` проверяет MIME (только image/*), размер (≤10MB), сохраняет файл с UUID-именем, записывает метаданные в `incident_photos`.

### 8. Логирование SLF4J

SLF4J — фасад для логирования. Уровни: TRACE < DEBUG < INFO < WARN < ERROR. В `application.yml` настроен уровень DEBUG для пакета проекта. Логи пишутся в консоль и файл.

**Применение:** каждый сервис логирует ключевые события. При превышении порога датчика — `log.warn()`. При критических ошибках — `log.error()`.

### 9. Swagger/OpenAPI

Swagger UI предоставляет интерактивную документацию API. `@Operation`, `@Tag`, `@Parameter` добавляют описания. `@SecurityScheme` позволяет вводить JWT прямо в интерфейсе.

**Применение:** доступен по http://localhost:8080/swagger-ui/index.html. Все 30+ endpoint'ов задокументированы.

### 10. Docker и контейнеризация

Docker — платформа контейнеризации. `docker-compose.yml` описывает мультиконтейнерное приложение. Namespaces и cgroups обеспечивают изоляцию процессов.

**Применение:** PostgreSQL запускается через `docker compose up -d`. Это гарантирует одинаковое окружение на любой машине.

### 11. Интеграция с Telegram

Telegram Bot API позволяет отправлять сообщения через HTTP POST на `https://api.telegram.org/bot{token}/sendMessage`. Токен получается от BotFather.

**Применение:** при создании тревоги `TelegramService.sendAlert()` вызывается асинхронно (`@Async`), чтобы не блокировать ответ API. Результат сохраняется в `telegram_logs`.

### 12. Отчёты PDF и XLSX

Apache POI создаёт Excel-файлы (.xlsx). OpenPDF создаёт PDF-документы. Данные выгружаются из БД за указанный период.

**Применение:** `ReportService` генерирует отчёты по тревогам. `ReportLog` фиксирует каждый факт генерации.

### 13. Многопоточность и асинхронность

`@EnableAsync` + `@Async` выполняют методы в отдельном потоке из пула. Это предотвращает блокировку основного потока при долгих операциях (HTTP-запрос к Telegram).

**Применение:** Telegram-уведомление не задерживает ответ API при создании тревоги.

## Заключение

В ходе выполнения курсового проекта была создана полноценная система мониторинга пожарной безопасности на основе Spring Boot 3. Реализованы все ключевые концепции курса:

- **Кроссплатформенность**: JVM позволяет запускать приложение на Windows, Linux и macOS без изменений кода
- **Архитектура**: чёткое разделение на слои (controller → service → repository)
- **Безопасность**: JWT, BCrypt, ролевая модель с granular правами
- **Интеграции**: PostgreSQL, Telegram Bot, CSV/PDF/XLSX
- **DevOps**: Docker Compose для воспроизводимого окружения
- **Качество**: unit-тесты с Mockito, структурированное логирование

Проект демонстрирует практическое применение изученных технологий и может служить основой для реальной системы безопасности учебного заведения.
